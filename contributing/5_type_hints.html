Type Hints

All arguments should use full and thorough type hints.

For the purposes of explanation, we will use three randomly chosen functions as examples: ivy.tan, ivy.roll and ivy.add

All functions which consume at least one array and also return at least one array can also consume and return ivy.Container instances in place of the arrays. Because of this, we refer to these functions as as flexible functions. These functions can also accept combinations of arrays and ivy.Container instances, broadcasting the arrays to the leaves of the containers. All flexible functions are also implemented as instance methods on both the ivy.Array and ivy.Container classes, with the first array argument replaced with self in both cases.

Furthermore, all array arguments can either be passed an ivy.Array or an ivy.NativeArray instance (i.e. torch.Tensor, np.ndarray etc. depending on the backend). However, ivy.NativeArray instances are not permitted in the output. Arrays are always returned as ivy.Array instances. Among other reasons, this is to ensure that operators (+, -, *, / etc.) performed on the array are framework-agnostic. We could not guarantee this if returning ivy.NativeArray instances, as we would have no control over the relevant special methods (__add__, __sub__, __mul__, __div__ etc.).

ivy.NativeArray instances are also not permitted for the out argument, which is used in many functions. This is because the out argument dicates the array to which the result should be written, and so it effectively serves the same purpose as the function return when no out argument is specified.

All arguments should be added on a new line, and the return type hint should also be added on a new line.

Taking all of these points into consideration, let’s take a look at the type hints for ivy.tan:

def tan(
    x: Union[ivy.Array, ivy.NativeArray, ivy.Container],
    out: Optional[Union[ivy.Array, ivy.Container]] = None
) -> Union[ivy.Array, ivy.Container]:

Because tan consumes one array and also returns one array, it is a flexible function. Therefore, both inputs x and out and the return are all permitted to be ivy.Container instances. Additionally, in alignment with the above explanations, x permits ivy.NativeArray instances, but out and the function return do not.

Similarly, the type hints for ivy.roll

def roll(
    x: Union[ivy.Array, ivy.NativeArray, ivy.Container],
    shift: Union[int, Tuple[int, ...]],
    axis: Optional[Union[int, Tuple[int, ...]]] = None,
    out: Optional[Union[ivy.Array, ivy.Container]] = None,
) -> Union[ivy.Array, ivy.Container]:

and ivy.add

def add(
    x1: Union[ivy.Array, ivy.NativeArray, ivy.Container],
    x2: Union[ivy.Array, ivy.NativeArray, ivy.Container],
    out: Optional[Union[ivy.Array, ivy.Container]] = None,
) -> Union[ivy.Array, ivy.Container]:

both also follow the same pattern.
